## Anti-Patterns Verdict

**Mostly clean.** Avoids gradient text, glassmorphism, purple neobank palette, hero-metric template.

Two fingerprints remain: (1) identical bg-ink banner on 3 pages; (2) magic-string `text-[11px] font-medium text-paper/45` copy-pasted 6 times across 3 files.

Deterministic scan: exit 0, completely clean.

## Priority Issues

### [P1] Instant irreversible deletion
handleDelete in transactions.tsx:104 calls setTransactions(transactions.filter(...)) directly on click. No confirmation, no undo, no toast. Mobile button is 28×28px (WCAG 2.5.5 failure). Fix: 5-second undo toast or inline confirm/cancel pair.

### [P1] Modifier only edits the amount
InlineEdit (transactions.tsx:397) exposes only montant. Wrong title/category/date requires delete + re-enter. Fix: add edit mode to the existing Modal with pre-populated fields.

### [P2] No filter state visibility
Active filter + tab switch = empty result that looks like missing data. No badge, no reset link near empty state. Fix: active-filter badge + "Réinitialiser les filtres" link. ~10 lines.

### [P2] Analytics dark banner is structural filler
bg-ink banner restates info already in the charts above it, using the most dominant visual treatment for redundant content. Fix: remove or move to top as genuine orientation layer.

### [P2] Duplicate aria-label on nav elements
Both desktop sidebar and mobile bottom nav use aria-label="Navigation principale". Screen readers cannot distinguish them. Fix: one-line change.

## Persona Red Flags

**Alex (power user):** Focus dropped to body after modal close. Date defaults empty, errored field not auto-focused. Both quick-entry buttons use same icon.

**Sam (screen reader):** No aria-live success announcement after modal close. Category select has no aria-required. subscriptions.tsx:201 labels prochainPaiement as "Dernier paiement le" — semantic error read aloud.
