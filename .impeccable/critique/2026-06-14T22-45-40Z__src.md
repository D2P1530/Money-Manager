## Anti-Patterns Verdict

## Overall Impression

The visual language is cohesive and disciplined. The real gap is discoverability and recovery: users with no prior context won't know where to start, users with real data volumes can't find transactions efficiently, and users who make mistakes have no safety net.

## What's Working

1. **Numbers-first typographic system.** IBM Plex Mono + tabular-nums on all monetary values, text-4xl for KPI numbers — the hierarchy signals "ledger" instantly.
2. **Semantic accessibility foundations.** aria-label on all nav elements, aria-live regions, scope="col" on table headers, caption on tables, sr-only for icon-only buttons.
3. **Design system coherence.** OKLCH token system (paper → surface → sunken → ink) used consistently. The bg-ink slabs serve real purposes, none decorative.

## Priority Issues

### [P1] No text search in transactions
**Why it matters**: With 50+ entries, filter-only approach requires recalling exact category names. A user looking for a specific merchant has no direct path.
**Fix**: Add text input to filter bar searching across titre and description fields.
**Suggested command**: /impeccable craft recherche transactions

### [P2] New-user / empty-state experience is absent
**Why it matters**: Fresh user sees CHF 0.00 everywhere and an empty chart — no guidance on what to do first.
**Fix**: When transactions.length === 0, show guided empty state with one CTA pointing to Transactions page.
**Suggested command**: /impeccable onboard

### [P2] "Saisie rapide" navigates away from dashboard
**Why it matters**: Clicking quick-entry triggers page navigation to /transactions then opens a modal — user loses dashboard context, 5 steps for what should be 2.
**Fix**: Open modal directly on dashboard page, remove navigate() calls.
**Suggested command**: /impeccable shape saisie rapide en modal sur le dashboard

### [P2] Bank balance input silently discards invalid input
**Why it matters**: Typing "1'200" (Swiss apostrophe separator) is silently ignored. No error feedback. Critical in Swiss context.
**Fix**: Show error message on invalid input; normalize apostrophe separators before parsing.
**Suggested command**: /impeccable harden

### [P2] Deletion is permanent with no recovery window
**Why it matters**: Confirmation shows only titre, not amount or date — could confirm the wrong "Courses". No undo after confirmation.
**Fix**: (a) Show full transaction detail in confirmation modal. (b) 5-second undo toast after delete.
**Suggested command**: /impeccable harden

## Persona Red Flags

**Alex (Power User)**: No Ctrl+N shortcut. No text search. No bulk delete. Saisie rapide page-navigates instead of opening in-place. High friction for daily use.

**Sam (Accessibility)**: Rapprochement grid uses <p> tags not dt/dd — screen reader loses column structure. Écart status is color-only (text-positive-soft / text-negative-soft) with no aria-label — WCAG 1.4.1 violation.

**Casey (Mobile)**: Filter bar on transactions requires scrolling past it to reach the table; unlikely to be discovered. AlignLeft description button in title cell has no explicit size — mis-tap risk.
